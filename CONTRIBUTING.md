# Guía de Contribución - WawaKalu

¡Gracias por tu interés en contribuir a WawaKalu! Este es un proyecto colaborativo diseñado para crear una aplicación educativa para niñas y niños.

## 🎯 Objetivo del Proyecto

WawaKalu es una aplicación web educativa e interactiva dirigida a niñas y niños para:
- Aprender conceptos básicos (números, letras, formas, colores)
- Desarrollar habilidades creativas
- Divertirse mientras aprenden
- Crear una experiencia inclusiva y accesible

## 🌟 Cómo Contribuir

### 1. Ideas y Sugerencias

- Abre un **Issue** para proponer nuevas actividades o mejoras
- Describe claramente tu idea y cómo beneficiaría a los niños
- Incluye ejemplos o referencias si es posible

### 2. Reportar Problemas

- Si encuentras un error, abre un **Issue** con:
  - Descripción del problema
  - Pasos para reproducirlo
  - Comportamiento esperado vs. actual
  - Capturas de pantalla si aplica

### 3. Agregar Nuevas Características

#### Actividades Sugeridas
- Juegos de memoria
- Rompecabezas interactivos
- Historias animadas
- Ejercicios de matemáticas básicas
- Canciones infantiles
- Actividades de música
- Juegos de palabras

#### Antes de Comenzar
1. Verifica que la característica no esté ya en desarrollo
2. Abre un Issue para discutir tu propuesta
3. Espera comentarios antes de empezar a codificar

### 4. Código

#### Estructura del Proyecto
```
WawaKalu/
├── index.html        # Página principal
├── styles.css        # Estilos y diseño
├── script.js         # Lógica e interactividad
├── README.md         # Documentación principal
└── CONTRIBUTING.md   # Esta guía
```

#### Estándares de Código

**HTML:**
- Usar etiquetas semánticas
- Mantener accesibilidad (alt text, ARIA labels)
- Código limpio y bien indentado

**CSS:**
- Usar clases descriptivas
- Mantener diseño responsive
- Colores brillantes y amigables para niños
- Animaciones suaves y no excesivas

**JavaScript:**
- Comentar código complejo
- Funciones pequeñas y específicas
- Nombres de variables descriptivos
- Validar inputs de usuario

#### Consideraciones Especiales

🎨 **Diseño:**
- Colores brillantes y contrastantes
- Fuentes grandes y legibles
- Botones grandes fáciles de presionar
- Íconos y emojis para mejor comprensión

👶 **Contenido apropiado:**
- Lenguaje simple y claro
- Mensajes positivos y alentadores
- Sin contenido violento o inapropiado
- Inclusivo para todas las niñas y niños

♿ **Accesibilidad:**
- Soporte para teclado
- Alto contraste
- Compatible con lectores de pantalla
- Funciona en dispositivos móviles

### 5. Proceso de Pull Request

1. **Fork** el repositorio
2. Crea una **rama** descriptiva: `git checkout -b feature/nueva-actividad`
3. Haz tus cambios siguiendo los estándares
4. **Prueba** tus cambios en diferentes navegadores/dispositivos
5. **Commit** con mensajes claros: `git commit -m "Agrega juego de memoria"`
6. **Push** a tu fork: `git push origin feature/nueva-actividad`
7. Abre un **Pull Request** con:
   - Descripción clara de los cambios
   - Screenshots o videos si aplica
   - Referencia a Issues relacionados

### 6. Testing

Antes de enviar tu PR, verifica:
- [ ] La aplicación se carga sin errores
- [ ] Todas las actividades funcionan correctamente
- [ ] El diseño se ve bien en móvil y desktop
- [ ] No hay errores en la consola del navegador
- [ ] El código sigue los estándares del proyecto
